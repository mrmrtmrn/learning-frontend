# 学習メモ

## [JavaScript Primer](https://jsprimer.net/) を利用してJavaScriptの基礎を学習しなおし中
- データ型とリテラル
  - データ型
    - プリミティブ型：immutable（変更不可）
      - Boolean
      - Number
      - BigInt
      - String
      - undefined
      - null
      - Symbol
    - オブジェクト
      - プリミティブ型以外のデータ
  - 型を調べたいときは`typeof`演算子
  - リテラル
    - 値を直接記述する方法
    - Stringでいうところの"Hello"とか、Integerでいうところの123とか、Booleanでいうところのtrue/falseとか
    - 数値リテラルは`2^53 - 1 つまり 9007199254740991`まで表現できる
    - それより大きい数値は`BigInt`型を利用する、宣言の仕方は数値の末尾に`n`をつける（例：`9007199254740992n`）
    - JavaScriptではダブルクォートとシングルクォートの意味はどちらも同じ。改行は`\n`で表現する。
    - テンプレートリテラルなら改行もそのまま記述できる。バッククォート（`` ` ``）で囲む。
    - 変数の挿入もできる。`` `${変数名}` ``の形で記述する。Rubyの`"#{}"`みたいな感じ。
    - オブジェクトリテラルは`{}`で囲む。`obj = { "key": "value" }`のように記述する。
      - 参照するときは`obj.key`または`obj["key"]`のように記述する。
    - 正規表現リテラルはスラッシュで挟む。例：`/\d+/`
- 演算子
  - べき乗演算子：`**`。例：`2 ** 3`は`8`
  - 単項プラス演算子：`+`。数値に変換する。例：`+"42"`は数値の`42`, `+"文字列"`は`NaN`
  - 単項マイナス演算子：`-`。数値に変換して符号を反転する。例：`-"42"`は数値の`-42`
  - 厳密比較演算子：`===`は型と値の両方を比較する。オブジェクトの場合は参照が同じである場合にtrueを返す。
  - 厳密不等価演算子：`!==`は型または値が異なる場合にtrueを返す。
  - 等価演算子：`==`は同じデータ型の場合は`===`と同じ挙動。異なるデータ型の場合は暗黙的型変換を行うので`1 == "1"`はtrueになる。
  - 不等価演算子：`!=`は暗黙的型変換を行う。極力使用を避けるべき。
  - falsyな値(暗黙的な型変換でfalseになる値)
    - `false`
    - `0`
    - `0n`（BigIntの0）
    - `""`（空文字列）
    - `null`
    - `undefined`
    - `NaN`
  - !!falsyでboolean型のfalseになる。!!truthyでboolean型のtrueになる。
  - Nullish coalescing演算子：`??`。左オペランドが`null`または`undefined`の場合に右オペランドを返す。
    - 例：`let result = value ?? "default"`は、`value`が`null`または`undefined`の場合に`"default"`を返す。
- 暗黙的な型変換
  - 暗黙的型変換でNaNが発生するケースはかなり厄介なので注意が必要
  - Booleanを使った型変換は正確人真偽値を得るための方法ではないので他の方法で解決できないか検討する
